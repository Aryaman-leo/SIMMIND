Week plan — SimMind · 3–4 hrs/day
Day 1 — Mon Project setup + Agent model 3 hrs
You end today with two AI agents that have personalities and can hold a belief.
  • Set up repo, venv, install langchain, fastapi, networkx, pydantic
  • Write agent.py — Agent dataclass with personality, beliefs, trust list
  • Write mock_mind.py — fake LLM that returns hardcoded JSON (zero API cost)
  • Write a test script: create 2 agents, print their profiles
✓ End of day win: python test_agents.py prints two distinct human profiles
Day 2 — Tue Conversation engine + belief update 4 hrs
Two agents can now talk about an idea and change their minds.
  • Write mind.py — LangChain prompt template for agent conversation
  • Prompt returns structured JSON: belief_delta, response_text, reasoning
  • Wire mock LLM into mind.py so it runs without API key
  • Test: inject idea "AI will take all jobs" → agent A talks to agent B → beliefs update
✓ End of day win: one conversation runs, belief score changes, reasoning prints
Day 3 — Wed Simulation tick loop + social graph 4 hrs
A full population runs for N ticks. You can watch beliefs spread in the terminal.
  • Write presets.py — 3 population presets (small town, polarised, social media)
  • Write simulation.py — tick loop, picks agent pairs from NetworkX graph
  • Save belief snapshots to SQLite after each tick
  • Run 20-agent simulation for 10 ticks, print belief adoption % per tick
✓ End of day win: terminal shows idea spreading tick by tick across 20 agents
Day 4 — Thu FastAPI backend + real LLM swap 4 hrs
Backend is live. You switch from mock to real LLM for the first time.
  • Write main.py — FastAPI routes: POST /simulate, GET /results/{sim_id}
  • WebSocket endpoint that streams tick updates to clients
  • Swap mock_mind for real LangChain + Claude Haiku (get free API key today)
  • Run one real simulation — watch actual LLM reasoning come back
✓ End of day win: curl the API, get real AI-generated belief updates back
Day 5 — Fri Idea mutation tracker 4 hrs
The differentiator. Ideas visibly mutate as they pass through different minds.
  • Extend mind.py prompt — LLM also returns mutated_idea_text when passing on belief
  • Store mutation chain per agent: original → version they heard → version they spread
  • Add /mutation-tree endpoint returning the idea's evolution as a tree structure
  • Print the mutation tree to terminal — see how idea text drifts across population
✓ End of day win: print shows "AI takes jobs" → "AI affects some jobs" → "AI helps jobs"
Day 6 — Sat Frontend — live network graph 4 hrs
The demo moment. Nodes glow as beliefs spread. You can show this to anyone.
  • Create React app, install D3.js and recharts
  • Force-directed graph — nodes = agents, colour intensity = belief adoption (gray → amber)
  • Connect to WebSocket — graph updates live as simulation ticks
  • Sidebar: adoption % line chart over time, top 5 most-influential agents
✓ End of day win: open browser, start sim, watch idea visually spread across the network
Day 7 — Sun Polish + deploy + README 3 hrs
Ship it. Live URL. Portfolio-ready README. Demo GIF recorded.
  • Deploy backend to Railway (free tier) — takes ~20 mins
  • Deploy frontend to Vercel — takes ~10 mins
  • Record a 60-second Loom demo: inject idea, watch it spread, show mutation tree
  • Write README: problem statement, architecture diagram, how to run, live demo link
✓ End of day win: live URL + GitHub repo + demo video = portfolio gold
If you fall behind: Day 5 (mutation) is a bonus — skip it and the project still stands. Days 1–4 backend + Day 6 frontend = complete project.
